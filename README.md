Перегрузка оператора =
Я узнал для себя новое, что если делаем присваивание a = b, где ранее была выделена память под a и b, при попытке очистить память может возникнуть баг, потому что два объекта начинают указывать на одну и ту же память.
Перегрузка оператора присваивания:
Существует встроенная функция класса: Класс & operator = (параметры),
вызов функции для примера: a.operator=(b);
Когда используем функцию operator=, нужно корректно управлять памятью, иначе могут возникнуть баги:
Удалить старую память, на которую указывал объект a
Выделить новую память
Скопировать данные из b в a
Вернуть ссылку на текущий объект 

Перегрузка операторов == и !=
Оператор должен возвращать какое-то логическое значение.
Первый пример: bool operator==(const Класс& ) действие 
Второй пример: bool operator!=(const Класс&) действие
Принцип работы:
Оператор == сравнивает объекты и возвращает true, если они равны
Оператор != Неравенство, сравнивает объекты и возвращает false, если они равны и наоборот
